# Gimbal Studio「专注控制盘」模式设计（2026-07-22）

**状态：** 已获对话确认  
**位置：** `tools/gimbal_studio/`  

---

## 1. 目标

提供菜单入口，将主窗口缩小为**仅含二维控制盘**的小窗；盘角保留唯一退出控件，再点即可恢复完整界面。

### 成功标准

- 菜单栏「文件」旁有新栏「视图 → 专注控制盘」
- 进入后：菜单栏、顶栏、Tab 均隐藏；窗口缩小；只见控制盘 + 角落「退出专注」按钮
- 拖盘发送逻辑与完整界面一致（同一 `PadWidget` 实例）
- 退出后：控件显隐与窗口几何（含最大化）恢复

---

## 2. 交互与状态

**状态：** `full`（默认）↔ `pad_only`

| 动作 | 行为 |
|------|------|
| 进入 | 保存 `geometry` + 是否最大化；隐藏 menuBar / topBar / tabs；显示专注容器；窗口缩至约 360×360；保留系统标题栏 |
| 退出 | 点盘角「退出专注」（主路径）；恢复显隐与几何；取消菜单勾选 |
| 关闭窗口 | 专注态下直接关窗即退出应用，不强制先恢复 |

**不做：** 置顶、无边框、独立第二窗口、快捷键、持久化「上次是否专注」。

---

## 3. 结构

逻辑集中在 `MainWindow`。

- checkable `QAction`「专注控制盘」挂在「视图」菜单
- `pad_focus_host`：中央容器 = 同一 `PadWidget` + 角落退出按钮
- 进入时将 `control_page.pad` **reparent** 到 host；退出时 reparent 回控制页原位
- 状态：`_pad_focus`、`_saved_geometry`、`_saved_maximized`

不改 HID/串口发送路径；未连接也可进入专注。

---
