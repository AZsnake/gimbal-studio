# 控制页「响应速度」预设挡位

日期：2026-07-22  
范围：`tools/gimbal_studio` 控制页 UI

## 目标

将控制页「Time」改为「响应速度」：提供五档预设按钮，并保留 ms 数字输入。协议 `Txxxx` 语义不变（到位时间 ms，数值越小越跟手）。

## 交互

| 控件 | 行为 |
|------|------|
| 标签 | `响应速度` |
| 预设按钮 | 极速 50 / 快速 100 / 标准 200 / 平滑 500 / 慢速 1000；互斥选中 |
| ms 框 | `QSpinBox` 0–9999，后缀 ` ms` |

同步：
- 点预设 → 写入对应 ms，按钮选中
- 手改到预设值 → 对应按钮选中
- 手改到非预设值 → 全部按钮取消选中
- 启动默认：平滑 **500 ms**

## 数据流

- 唯一真相仍是 `time_spin` 的 ms 值
- 拖动发送、写入动作组、`groups_page.set_current_pose` 继续读该值
- `apply_pose` 带回 time 时更新数字框，按钮按同步规则更新
- 不改协议 / ini；不持久化上次挡位

## 实现

- 方案：控制页内联改造（不抽独立 widget）
- 主要文件：`ui/control_page.py` + `tests/test_control_page.py`

## 非目标

- 速度倒序映射、记住上次挡位、改 groups 表格 UI、改 HID/协议层
