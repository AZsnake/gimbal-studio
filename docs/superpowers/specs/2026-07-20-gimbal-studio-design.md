# Gimbal Studio 设计规格

**日期：** 2026-07-20  
**状态：** 已批准（对话确认）  
**位置：** `tools/gimbal_studio/`  
**目标：** 用开源、跨平台的 PySide6 程序，替代 Windows 专用 Zide，通过串口控制众灵 ZL-IS2 及现有双轴云台。

---

## 1. 背景与目标

### 1.1 现状

- 仓库 `STABLIZER` 含 FPGA / 相机资料，以及 `docs/路舵机控制器资料/` 中的众灵 24 路舵机控制器（ZL-IS2）与 **Zide V5.99**。
- 云台工程配置见 `docs/路舵机控制器资料/003-软件工具/001-图形化上位机软件/config_bes.ini`：启用通道为 **水平（id=0）**、**倾斜（id=1）**，PWM 范围 500–2500，并含扫描演示动作组 G0000–G0019。
- Zide 仅 Windows；功能面远超两轴云台日常需求（手柄/红外/映射/IO/固件升级等）。

### 1.2 成功标准

- 能在 Windows / macOS / Linux 上通过串口连接控制板，实时控制云台。
- 能打开、编辑、保存与 Zide 兼容的 `.ini`（至少兼容 `config_bes.ini`）。
- 支持动作组在线执行、脱机调用（`$DGT`）、下载到板、急停。
- 界面美观、结构清晰；代码开源可维护。
- 提供源码运行方式与 PyInstaller 打包脚本。

### 1.3 非目标（首版明确不做）

- HID 连接
- 手柄 / 红外 / 映射 / 按键 / IO / 语音等外设配置页
- 固件升级
- 完整复刻 Zide 全部菜单与机器人皮肤

## 2. 决策摘要

| 项 | 选择 |
|----|------|
| 功能范围 | 云台精简版（实时控制 + 动作组 + `.ini` + 在线/脱机/下载） |
| 连接 | 仅串口（默认 115200 8N1） |
| UI 框架 | PySide6 |
| 双轴操作 | 二维盘 + 滑条/数值框 |
| 通道数 | 读 `.ini` 中 `enable=true` 动态生成；二维盘绑定前两个启用通道 |
| 主布局 | 标签分页：控制 / 动作组 / 串口日志 |
| 视觉 | 石墨琥珀（近黑底 + 琥珀强调） |
| 架构 | 分层单体（protocol / serial_io / project / ui） |
| 交付 | 源码 + `scripts/pack.py`（PyInstaller） |

## 4. 界面与交互

### 4.1 整体

- 应用名：**Gimbal Studio**（顶栏品牌级主信号，不被次要文案压过）。
- 主题：**石墨琥珀** — 近黑/石墨背景、琥珀强调色、清晰层级；避免紫白渐变与奶油衬线套路。
- 顶栏全局：端口列表、刷新、波特率、连接/断开、连接状态指示。

### 4.2 页「控制」

- 左侧：大号二维盘（X=水平通道，Y=倾斜通道；即前两个 `enable=true` 的通道）。
- 右侧：每个启用通道的标题、滑条、PWM 数值框；尊重 `pmin`/`pmax`/`bias`（bias 在显示/下发策略中与 Zide 语义对齐：偏差用于组装校准，首版至少读写保留，控制时应用方式与手册一致）。
- 公共控件：Time、归中（PWM=1500）、急停（`$DST!`）。
- 二维盘与滑条共用「当前姿态」状态，双向同步。
- 拖动时节流发送，避免串口洪水。
- 方向键微调：首版实现。

### 4.3 页「动作组」

- 表格：组号 Gxxxx、各通道 PWM/Time、可选备注。
- 增组 / 插组 / 删组；复制 / 剪切 / 粘贴。
- 单击选中并同步到控制页姿态；双击立即将该组指令下发（在线预览）。
- 起始 / 终止 / 次数（0=循环）。
- **在线执行**：按序发送各组指令串，组间按 Time 等待，可取消。
- **脱机执行**：发送 `$DGT:start-end,count!`。
- **下载动作组**：从选中组起写入板内存储，可中途停止；帧格式以实现阶段对照手册附录与真机确认。
- **设/取消开机动作**：对应板端指令。
- 文件：打开 / 保存 Zide 兼容 `.ini`。

### 4.4 页「串口日志」

- 发送框、接收框、清空。
- 支持手写原始 ASCII 指令调试。

## 5. 协议与数据流

### 5.1 核心指令（ASCII）

来源：ZL-IS2 手册附录与示例代码。

| 用途 | 格式 | 示例 |
|------|------|------|
| 单舵机运动 | `#IndexPpwmTtime!` | `#000P1500T1000!` |
| 多舵机组合 | `{#000P…!#001P…!}` | 动作组一行 |
| 调用单组 | `$DGS:n!` | `$DGS:0!` |
| 调用区间 | `$DGT:start-end,count!` | `$DGT:0-10,1!`（count=0 循环） |
| 停止 | `$DST!` / `$DST:x!` | 全部或指定 |
| 复位等 | `$RST!` 等 | 按需 |

Index 为 3 位（000–254）；pwm 4 位（0500–2500）；time 4 位毫秒（0000–9999）。

### 5.2 实时控制流

```
UI 变更 → protocol.build_move(...) → serial 写队列 → 板执行
板回包 → 读线程 → 队列/信号 → 日志页刷新
```

写操作串行化。

### 5.3 工程文件（`.ini`）

兼容 Zide 工程关键段：

- `[global]`：版本等信息（可读可写，不强制依赖）
- `[steerN]`：`title`、`id`、`pmin`、`pmax`、`bias`、`labLeft`/`labRight`、`enable`、`x`/`y`（坐标可保留，首版可不用于布局）
- `[group]`：`Gxxxx={Gxxxx#000P…T…!…}`
- `[cellName]` / `[cellCount]`：动作组合/折叠元数据（能读则保留；首版 UI 可不做折叠组合，但不丢字段）

默认可打开仓库内 `config_bes.ini` 作为验收样例。

首版不要求实现 `[ps2_config]`、`[ir_config]`、`[LD*]`、`[CY*]`、`[btn_config]` 等外设配置的编辑 UI；读写策略：**未知段原样保留**，避免破坏完整工程文件。

### 5.4 下载与开机动作

- 下载：从选中组到末组依次写入；实现时以手册「下载动作组」行为与真机抓包/文档为准，封装在 `protocol` + `serial_io` 的下载会话中。
- 开机动作：起始终止次数与设/取消命令封装为明确 API；具体字符串在实现阶段对照手册确认并写入测试。

## 6. 错误处理

- 端口打开失败 / 占用 / 意外断开：状态指示为断开，清空或暂停发送队列，提示重连；UI 不崩溃。
- PWM 越界：滑条钳位；数值框拒绝并提示。
- 未连接时执行/下载：按钮禁用或明确提示。
- 长任务（在线执行、下载）：可取消；异常记入日志并恢复可操作状态。
- `.ini` 损坏或缺段：尽量容错加载，汇总警告；不静默丢弃已解析数据。

## 7. 测试与验收

### 7.1 自动化

- `protocol`：编解码往返、边界（pwm/time 位数填充）。
- `project`：对 `config_bes.ini` 读→写→再读，关键字段一致；未知段保留。

### 7.2 手动验收

1. 枚举串口并连接 115200。
2. 控制页拖动二维盘与滑条，云台响应；急停有效。
3. 打开 `config_bes.ini`，通道与动作组正确。
4. 在线执行一段扫描组。
5. 下载后脱机 `$DGT` 执行；断电上电后脱机仍可用（若板支持存储）。
6. 日志页收发原始指令正常。
7. Windows 与至少另一平台（或 Linux CI 逻辑测试 + 本机一平台）源码可运行；打包脚本可生成可执行文件（允许仅在主开发平台验证打包）。

### 7.3 CI

仅跑无硬件单元测试；GUI / 真机不进 CI。

## 9. 模块职责一览

| 单元 | 做什么 | 怎么用 | 依赖 |
|------|--------|--------|------|
| `protocol` | 指令字符串构建/解析 | 纯函数 API | 无 |

## 10. 风险与实现注意

- **下载帧细节**：公开手册对「下载动作组」的逐字节过程不如单指令清晰，需真机验证；规格允许实现阶段小幅调整，但不改变用户可见行为（从选中组下载、可停）。
- **Bias 语义**：总线设备与 PWM 云台不完全相同；本项目云台为 PWM，以 `config_bes.ini` 中 bias 字段保留为主，若板端忽略 bias 则 UI 仍显示但不强制下发特殊指令。
- **跨平台串口权限**：Linux 可能需用户加入 `dialout`；README 说明。
